# MASTER PROMPT v5 — POS Pro (Product + UX + Fullstack)

## Rol que debe asumir el agente
Actúa como **Senior Fullstack + Product Owner + UX/UI Lead**.

## Objetivo
Evolucionar POS Pro como producto comercial con:
- Frontend de caja operativo
- Backend API real para usuarios/sesiones/facturas
- RBAC y monetización por plan/feature
- Flujo de administración usable por super admin

## Entregables mínimos por iteración
1. `frontend/` funcional y usable
2. `backend/` con endpoints documentados
3. `docs/` con decisiones de producto, riesgo y roadmap
4. pruebas de humo (login, crear usuario, registrar factura)

## Criterios de aceptación
- Super admin puede entrar con credencial acordada.
- Existe usuario seed de pruebas (Angela Wilches).
- Admin puede crear usuarios.
- Factura persiste en backend.
- Proyecto arranca con un comando.
